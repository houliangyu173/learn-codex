# 项目产品文档

## 1. 项目定义

### 1.1 项目名称

BookHub - 在线电子书阅读平台

### 1.2 项目目标

构建一个支持多来源书籍榜单、详情与阅读入口整合的平台，实现以下 V1 目标：

1. 支持从多种数据接入方式获取书籍榜单与详情数据。
2. 对书籍进行自动分类和来源标识管理。
3. 提供书籍搜索、浏览、详情和阅读入口能力。
4. 提供后台同步、上下架、日志查看和书架能力。

## 2. 数据接入原则

项目不预设固定技术接入方式，可使用公开 API、授权接口、页面采集、人工导入等方式接入数据，但必须满足以下原则：

1. 数据来源必须具备可使用依据或业务授权。
2. 对版权站点，默认只采集榜单和详情元数据，不默认站内分发正文。
3. 阅读能力按来源区分为站内阅读和站外跳转两类。
4. 若后续获得合作授权，可再扩展为站内正文阅读。

## 3. 技术范围

### 3.1 后端

1. JDK 1.8
2. Spring Boot 2.x
3. MyBatis-Plus
4. MySQL 8.0
5. Redis 预留
6. 定时任务采用 Spring `@Scheduled`

### 3.2 前端

1. Vue2
2. Vue Router
3. Axios
4. Element UI
5. V1 不引入 Vuex
6. 不使用 TypeScript

## 4. 系统模块

### 4.1 书籍采集模块

职责：

1. 支持按来源类型采集榜单、详情和阅读入口元数据。
2. 支持 API、页面采集和补充元数据编排。
3. 完成去重、落库与同步日志记录。
4. 支持按来源、榜单类型、关键字、数量触发手动同步。
5. 对 17K 来源，优先采集榜单和详情信息。
6. 对可公开阅读来源，继续保留站内阅读接入。

后台接口：

1. `POST /admin/book/sync`
2. `GET /admin/book/sync/log/list`

### 4.2 书籍管理模块

职责：

1. 分页查询书籍。
2. 按关键字、分类筛选书籍。
3. 查看书籍详情。

接口：

1. `GET /book/list`
2. `GET /book/{id}`
3. `GET /book/category/list`

### 4.3 自动分类模块

V1 采用关键词匹配策略。

分类规则：

1. `science -> 科学`
2. `history -> 历史`
3. `fiction -> 小说`
4. `computer -> 技术`
5. `children -> 儿童`

分类来源字段：

1. `title`
2. `subjects`
3. `bookshelves`
4. `description`

### 4.4 阅读模块

V1 阅读能力：

1. `INTERNAL`：站内 `html/txt` 阅读
2. `EXTERNAL`：跳转原站阅读

接口：

1. `GET /book/read/{id}`

### 4.5 17K 榜单接入模块

职责：

1. 采集 `www.17k.com` 男生作品点击榜、女生作品点击榜。
2. 采集作品详情页中的书名、作者、分类、简介、封面、状态和详情链接。
3. 将 17K 作品标记为站外阅读来源。
4. 在详情页和阅读页中提供前往原站阅读入口。

V1 处理方式：

1. 采集榜单页和详情页元数据。
2. 不在系统内存储或分发 17K 正文。
3. 阅读时跳转 17K 原站。

### 4.6 书架模块

职责：

1. 用户将书加入书架。
2. 用户查看书架列表。
3. 系统记录阅读进度。

接口：

1. `POST /bookshelf/add`
2. `PUT /bookshelf/progress`
3. `GET /bookshelf/list`

### 4.7 后台管理模块

职责：

1. 手动同步书籍
2. 查看同步日志
3. 控制书籍上架与下架
4. 输入同步条件并筛选日志
5. 区分来源类型和榜单类型执行同步

接口：

1. `POST /admin/book/sync`
2. `PUT /admin/book/status`
3. `GET /admin/book/sync/log/list`

## 5. 当前数据库模型

当前已落地的核心表包括：

1. `book`
2. `book_category`
3. `user_bookshelf`
4. `book_sync_log`

建议新增或扩展字段：

1. `read_mode`：阅读模式，`INTERNAL` / `EXTERNAL`
2. `external_detail_url`：外部详情页地址
3. `external_read_url`：外部阅读地址
4. `rank_type`：榜单类型，如 `17k_male_click`、`17k_female_click`
5. `rank_no`：榜单排名
6. `rank_value`：榜单值，如点击数

3. 管理员可手动触发同步。
4. 管理员可执行上下架。
5. 管理员可查看同步日志。
6. 用户可加入书架并更新阅读进度。
7. 17K 来源作品可展示榜单信息并跳转原站阅读。
