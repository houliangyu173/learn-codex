# 需求规格说明书

## 1. 需求目标

本轮围绕 BookHub V1 的核心业务增强，补齐真实采集、书架和后台日志三项能力。

## 2. 需求范围

### 2.1 真实采集流程

1. 后台同步不再仅写入样例数据。
2. 系统应调用 Gutendex API 获取真实书籍元数据。
3. 系统应在字段不足时按需使用 Open Library 作为补充来源。
4. 同步结果应继续写入 `book` 和 `book_sync_log`。
5. 当外部接口失败时，系统应给出明确日志和结果说明。

### 2.2 书架功能

1. 用户可将书籍加入书架。
2. 用户可查看书架列表。
3. 用户可更新阅读进度。
4. V1 无用户体系时，允许以默认用户ID方式先完成闭环。

### 2.3 后台日志增强

1. 日志查询支持分页。
2. 日志应补充来源、状态、触发方式、成功数、失败数、消息等字段。
3. 后台页面应支持查看最近日志，并预留筛选能力。

## 3. 验收标准

1. 触发同步后，数据库中新增真实来源书籍数据或记录明确失败日志。
2. 可通过接口或页面完成加入书架、查看书架、更新进度。
3. 后台日志展示信息比当前更完整，查询更稳定。
