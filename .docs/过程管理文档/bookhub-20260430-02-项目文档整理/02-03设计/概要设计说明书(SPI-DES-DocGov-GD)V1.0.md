# 概要设计说明书

## 1. 设计方案

项目级文档统一放在仓库根目录，按“总入口 + 产品 + 规范 + 进度”组织：

1. `README.md`：项目总入口
2. `项目产品文档.md`：产品定义与范围
3. `开发规范文档.md`：开发、联调、文档维护规则
4. `业务进度文档.md`：当前状态与下一步计划

## 2. 过程文档策略

项目级说明与过程文档分离：

1. 根目录文档负责长期稳定说明。
2. `.docs/过程管理文档/` 负责阶段性任务追踪。

## 3. 维护策略

后续每轮工作优先判断变更属于：

1. 产品范围
2. 技术规范
3. 进度状态
4. 阶段任务过程

并同步更新对应文档。
