# 需求规格说明书

## 1. 需求背景

当前项目已经进入多模块并行开发阶段，需要建立稳定的项目级说明文件，避免产品范围、技术约束和业务进度信息分散或过时。

## 2. 需求目标

1. 建立统一项目入口文档。
2. 沉淀产品范围说明。
3. 沉淀开发规范说明。
4. 沉淀业务进度说明。
5. 明确后续每次工作需同步哪些文档。

## 3. 交付要求

1. 删除或淘汰无用项目说明文件。
2. 新增根目录项目级说明文件。
3. 同步当前前后端、数据库和联调进展。
4. 将文档维护要求明确写入规范。
