# 需求规格说明书

## 1. 文档目的

本文档用于明确 BookHub V1 在线电子书阅读平台的业务范围、角色职责、业务流程、功能需求和约束条件，作为后续系统设计和代码实现的依据。

## 2. 项目概述

BookHub 是一个面向公开电子书资源的在线阅读平台，目标是构建一个后台系统，实现开放电子书数据采集、自动分类、前台查询浏览、在线阅读及后台运营管理。

## 3. 业务目标

1. 从开放电子书 API 获取合法公开的书籍数据。
2. 对采集到的书籍进行标准化清洗、去重与自动分类。
3. 提供书籍列表、详情和在线阅读能力。
4. 提供后台同步、日志查看和上下架管理能力。

## 4. 角色定义

### 4.1 管理员

负责触发书籍同步、查看采集日志、执行书籍上下架管理。

### 4.2 普通用户

负责浏览书籍列表、查看书籍详情、进入在线阅读、维护个人书架和阅读进度。

### 4.3 系统任务

负责按规则调用外部开放 API，完成数据采集、清洗、去重、分类和入库等自动化处理。

## 5. 核心业务流程

### 5.1 主流程

1. 管理员手动触发同步，或系统定时任务自动触发同步。
2. 系统调用 Gutendex API，按搜索词、语言、主题等参数拉取书籍元数据。
3. 系统按需调用 Open Library API 补充目录信息。
4. 系统解析返回数据，提取标题、作者、简介、封面、语言、阅读链接、下载链接等字段。
5. 系统根据 `source + source_book_id` 进行去重校验。
6. 系统基于 `title`、`subjects`、`bookshelves`、`description` 进行关键词分类。
7. 系统保存书籍和分类关系，并生成同步日志。
8. 管理员根据业务需要执行书籍上架或下架。
9. 前台用户仅能浏览已上架书籍，并查看详情与进入阅读。

### 5.2 用户阅读流程

1. 用户进入书籍列表页。
2. 用户按关键字或分类浏览书籍。
3. 用户查看书籍详情。
4. 用户点击阅读，系统根据书籍可读资源类型跳转到 TXT 或 HTML 阅读入口。
5. 用户可将书籍加入书架并记录阅读进度。

### 5.3 后台管理流程

1. 管理员发起手动同步。
2. 系统执行采集和分类。
3. 系统记录同步结果、成功数量、失败数量与错误原因。
4. 管理员查看同步日志。
5. 管理员对书籍执行上架或下架。

## 6. 功能需求

### 6.1 书籍采集模块

1. 支持对 Gutendex API 发起查询。
2. 支持 `search`、`language`、`topic` 参数。
3. 支持解析 JSON 响应。
4. 支持将外部字段映射为内部统一书籍模型。
5. 支持基于来源主键去重。
6. 支持记录同步日志。

### 6.2 自动分类模块

1. 支持基于关键词进行规则分类。
2. 规则至少覆盖以下映射：
   `science -> 科学`
   `history -> 历史`
   `fiction -> 小说`
   `computer -> 技术`
   `children -> 儿童`
3. 分类判断来源字段包括 `title`、`subjects`、`bookshelves`、`description`。
4. 未命中的书籍允许进入默认分类或待分类状态。

### 6.3 书籍管理模块

1. 支持分页查询书籍列表。
2. 支持按关键字、分类、语言进行筛选。
3. 支持查看书籍详情。
4. 仅展示状态为已上架的书籍。

### 6.4 阅读模块

1. 支持 TXT 在线阅读。
2. 支持 HTML 阅读入口。
3. 支持根据书籍资源类型返回正确阅读地址。
4. 不存储盗版正文，不提供非法资源。

### 6.5 后台管理模块

1. 支持手动同步书籍。
2. 支持查看采集日志。
3. 支持书籍上架与下架。

## 7. 业务规则

1. 数据来源必须限定为 Gutendex 和 Open Library。
2. 禁止爬取盗版电子书网站。
3. 禁止绕过 API 直接抓取 HTML 页面。
4. 禁止存储版权受限的正文内容。
5. 书籍前台展示必须受上架状态控制。
6. 分类结果应在书籍可见前完成，避免前台分类信息缺失。

## 8. 数据需求

### 8.1 书籍表

需保存标题、作者、描述、封面、来源、来源书籍编号、语言、分类、文件类型、阅读地址、下载地址、状态和创建时间等信息。

### 8.2 分类表

需保存分类名称、父级分类、排序等信息，用于前台分类筛选和后台管理。

### 8.3 书架表

需保存用户与书籍关系、阅读进度、最近阅读时间，用于个性化阅读体验。

## 9. 非功能约束

1. 技术栈固定为 JDK 1.8、Spring Boot 2.x、MyBatis-Plus、MySQL 5.7+。
2. 缓存能力预留 Redis。
3. 定时任务采用 Spring `@Scheduled`。
4. 系统分层必须遵循 `controller -> service -> domain -> mapper -> database`。
5. 禁止 `controller` 直接操作 `mapper`。
6. 禁止 `service` 编写 SQL。
7. 禁止 `entity` 与 `VO` 混用。

## 10. 验收标准

1. 管理员可成功触发书籍同步。
2. 系统可对外部书籍完成标准化处理和去重。
3. 系统可对书籍完成关键词分类。
4. 用户可查询已上架书籍并进入阅读。
5. 管理员可查看同步日志并控制书籍上下架。
