# 业务进度文档

## 1. 当前总体状态

项目当前处于 V1 持续联调阶段，前后端主链路已覆盖真实采集、阅读、书架、后台日志增强以及 17K 榜单首版接入。

## 2. 已完成内容

### 2.1 产品与设计

1. 已完成 BookHub V1 业务流程梳理。
2. 已完成基础需求规格和概要设计沉淀。
3. 已确定并落地 17K 榜单首版接入，采用“榜单/详情采集 + 原站跳转阅读”方案。

### 2.2 后端

1. 已完成书籍列表、详情、阅读信息接口。
2. 已完成分类列表接口。
3. 已完成手动同步、上下架接口。
4. 已完成同步日志分页查询接口。
5. 已完成 MyBatis-Plus + MySQL 8.0 持久化迁移。
6. 已完成数据库建表与基础数据初始化。
7. 已完成 Gutendex 真实采集接入，并为测试环境保留 mock 采集模式。
8. 已完成 Open Library 补充采集接入，用于数量补足和元数据补全。
9. 已完成书架接口与阅读进度写回。
10. 已完成同步日志筛选与详细字段记录。
11. 已完成 17K 榜单来源首版同步链路、外链阅读模式与基础数据模型扩展。
12. 已完成站内阅读代理修复，`Gutendex/Open Library` 来源的 `html/txt` 阅读改为走后端正文代理，避免前端跨域与外链 `iframe` 空白问题。

### 2.3 前端

1. 已完成首页、详情页、阅读页、后台页、书架页。
2. 已完成首页动态分类筛选。
3. 已完成后台同步条件输入、17K 榜单入口与日志展示增强。
4. 已切换前端开发端口到 `3000`。

## 3. 当前可联调范围

1. 首页可按关键字、分类查询书籍。
2. 详情页可查看书籍信息并进入阅读。
3. 阅读页可通过后端代理渲染 `html/txt`。
4. 阅读页进入后可自动写入基础阅读进度。
5. 书架页可查看列表并更新进度。
6. 后台可按条件触发真实同步并查看筛选日志。
7. 后台可执行上下架操作。
8. 17K 来源作品可展示榜单信息并跳转原站阅读。

## 4. 待办事项

### 4.1 高优先级

1. 补强 17K 生产环境详情页采集稳定性。
2. 增加 17K 榜单类型筛选与来源筛选。
3. 完善 17K 同步失败重试和限流保护。

### 4.2 中优先级

1. 丰富分类规则与默认分类策略。
2. 增加后台日志详情弹层与导出能力。
3. 增加更多集成测试与异常场景测试。
4. 丰富同步失败重试与限流保护。

## 5. 当前风险

1. 当前真实采集依赖 Gutendex 与 Open Library，外部 API 波动会直接影响同步效果。
2. 分类能力仍为基础规则版，复杂场景准确率有限。
3. 用户体系尚未建设，书架当前依赖默认用户ID。
4. 17K 页面结构变化可能影响榜单与详情采集稳定性。
5. 外部 HTML 阅读资源仍可能包含第三方样式、图片或相对路径差异，个别书籍的沉浸式排版体验仍需继续观察。

## 6. 下一个推荐动作

推荐优先顺序：

1. 17K 详情采集稳定性增强
2. 同步失败重试与限流保护
3. 分类规则增强
