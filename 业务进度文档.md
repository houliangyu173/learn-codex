# 业务进度文档

## 1. 当前总体状态

项目当前处于 V1 可联调阶段，前后端主链路已经打通，数据库初始化与基础样例数据已落地。

## 2. 已完成内容

### 2.1 产品与设计

1. 已完成 BookHub V1 业务流程梳理。
2. 已完成基础需求规格和概要设计沉淀。

### 2.2 后端

1. 已完成书籍列表、详情、阅读信息接口。
2. 已完成分类列表接口。
3. 已完成手动同步、上下架接口。
4. 已完成同步日志分页查询接口。
5. 已完成 MyBatis-Plus + MySQL 8.0 持久化迁移。
6. 已完成数据库建表与基础数据初始化。

### 2.3 前端

1. 已完成首页、详情页、阅读页、后台页。
2. 已完成首页动态分类筛选。
3. 已完成后台同步日志展示。
4. 已切换前端开发端口到 `3000`。

## 3. 当前可联调范围

1. 首页可按关键字、分类查询书籍。
2. 详情页可查看书籍信息并进入阅读。
3. 阅读页可渲染 `html/txt`。
4. 后台可触发同步并查看日志。
5. 后台可执行上下架操作。

## 4. 待办事项

### 4.1 高优先级

1. 将手动同步从样例数据写入升级为真实外部 API 采集。
2. 补充同步日志筛选条件与详情能力。
3. 将书架功能真正接入前后端。

### 4.2 中优先级

1. 丰富分类规则与默认分类策略。
2. 增加后台同步条件输入，如关键词、语言、主题。
3. 增加更多集成测试与异常场景测试。

## 5. 当前风险

1. 当前同步逻辑仍是演示实现，尚未真正对接 Gutendex / Open Library。
2. 分类能力仍为基础规则版，复杂场景准确率有限。
3. 用户体系尚未建设，书架表当前仅完成数据库准备。

## 6. 下一个推荐动作

推荐优先顺序：

1. 真实采集流程
2. 书架功能
3. 后台日志增强
