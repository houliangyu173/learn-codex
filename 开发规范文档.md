# 开发规范文档

## 1. 代码分层规范

后端必须遵循：

`controller -> service -> domain -> mapper -> database`

禁止：

1. `controller` 直接调用数据库访问逻辑。
2. `service` 直接编写 SQL。
3. `entity` 与前端展示对象混用。

## 2. 后端规范

### 2.1 技术规范

1. Spring Boot 2.7.x
2. JDK 1.8
3. MyBatis-Plus
4. MySQL 8.0

### 2.2 返回体规范

所有接口统一使用：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 2.3 数据库规范

1. 应优先维护 `src/main/resources/sql/` 下的初始化脚本。
2. 建表字段优先使用下划线命名。
3. Java 属性使用驼峰命名，由 MyBatis-Plus 做映射。
4. 新增业务表时，尽量同时补索引和初始化数据策略。
5. 多来源书籍场景必须显式区分来源渠道、阅读模式和外部链接字段。

## 3. 前端规范

### 3.1 技术规范

1. Vue2
2. Vue Router
3. Axios
4. Element UI

### 3.2 目录规范

```text
bookhub-web/src
├── api/
├── router/
├── utils/
└── views/
```

### 3.3 请求规范

1. 所有接口统一从 `src/utils/request.js` 发起。
2. 业务 API 统一收口到 `src/api/book.js`。
3. 页面组件中避免直接写散乱的 axios 请求。

## 4. 联调与端口规范

1. 后端默认端口：`8080`
2. 前端默认开发端口：`3000`
3. 前端通过代理转发 `/book`、`/admin`、`/bookshelf` 到后端。

## 5. 测试与验证规范

1. 后端改动后，优先运行相关测试，再跑全量测试。
2. 当前后端统一验证命令：

```bash
mvn -Dmaven.repo.local=/tmp/codex-m2 test
```

3. 前端改动后，至少运行一次：

```bash
cd bookhub-web
npm run build
```

## 6. 文档同步规范

以后每次工作默认同步以下文档：

1. 功能范围变化：更新 `项目产品文档.md`
2. 技术规则变化：更新 `开发规范文档.md`
3. 完成状态变化：更新 `业务进度文档.md`
4. 较大任务过程：更新 `.docs/过程管理文档/` 对应 TaskID 文档

## 7. 当前业务约定

1. 真实采集默认对接 Gutendex，并允许使用 Open Library 做补充采集与字段补全；测试环境通过 `bookhub.sync.mock-enabled=true` 关闭外网依赖。
2. 17K 类版权站点默认按 `EXTERNAL` 阅读模式处理，V1 仅采集榜单与详情元数据，并保留原站阅读链接。
3. 书架 V1 默认用户ID为 `1`，后续接入登录态时再迁移为真实用户上下文。
4. 同步日志必须记录 `requestParams`，失败场景应记录 `errorMessage`。

## 8. 当前约定

1. 项目级说明文件优先维护在根目录。
2. 过程追踪和阶段性记录维护在 `.docs/过程管理文档/`。
3. 过时说明文件应删除或合并，避免多个版本并行冲突。
